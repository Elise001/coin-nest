# 背景：
1. 市场上充斥着广告+收费模式
2. 记账APP核心是低成本、可持续、异常感知，大部分APP自动记账收费+交互复杂，学习成本高，违背记账初心
   + 低成本指的是学习成本低，交互模式简单，让用户花在APP上的时间减少甚至不用打开
   + 可持续指的是无负担场景下能做到长期坚持，也就是全自动记账能力+正向反馈机制
   + 异常感知是指智能识别异常支出，或短时间内大量支出记录
3. 支付宝、微信、银行卡都有消费记录，记账APP的优势在于提供统一的消费记录入口，和快速发现异常

## v1.4（智能分类 MVP）
1. 本地智能分类闭环（Local-First）：
   + 自动记账入库前先执行“规则分类 + 本地学习规则”融合判定。
   + 当规则分类结果为“待分类/自动识别”时，优先尝试本地学习规则自动补全分类。
   + 当本地学习规则命中次数达到阈值后，可覆盖通用规则结果，降低重复手动改类。
2. 本地学习机制：
   + 用户对自动记账流水进行分类纠正后，系统从通知备注中提取关键词并写入本地规则库。
   + 规则维度：交易类型（收入/支出）+ 渠道来源（支付宝/微信/银行卡等）+ 关键词。
   + 规则命中会累积命中次数，用于后续优先级排序（命中次数越高越优先）。
3. 数据可迁移：
   + 本地备份/导入新增智能分类规则数据，换机或重装后可延续已有学习效果。
4. 可解释与可控性增强：
   + 自动分类命中时会在备注追加 `[SMART:关键词#命中次数]` 标记，便于问题排查和用户理解分类来源。
   + 设置页新增“智能分类学习状态”卡片，可查看规则总数、高置信规则数与高频关键词。
   + 支持一键清空本地学习规则（不影响历史流水），用于重置学习偏差。


## v1.5（正向反馈与留存）
1. 连续记录体系（本地计算）：
   + 新增“当前连续记录天数”“历史最长连续天数”“本月活跃记录天数”三项留存指标。
   + 指标全部基于本地已确认流水计算，不依赖联网服务。
2. 轻量徽章机制（低打扰）：
   + 新增连续记录徽章阈值：3天、7天、14天、30天。
   + 达到阈值后在首页“成就与留存”卡片展示已解锁徽章。
3. 达成庆祝反馈（即时但克制）：
   + 当日连续记录达到里程碑阈值时，展示一句轻量庆祝文案，增强正反馈。
   + 不弹窗、不打断主流程，避免噪音和打扰。
